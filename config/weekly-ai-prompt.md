## 指示

状況を踏まえて**最も重要な3つの提案**を生成し、それぞれに適切な優先度（高・中・低）を割り当ててください。

**重要**: 同じ優先度が複数あっても構いません。
- 緊急課題が3つある場合 → 「高・高・高」でも可
- 緊急が2つ → 「高・高・中」などでも可
- 全体が順調な場合 → 「中・中・低」や「低・低・低」でも可

### 優先度の目安

- **高**: 納期・生産・クレームに直結し、**今日〜明日中**に手を打つべきこと
- **中**: **今週中**に実施すべき改善・フォロー
- **低**: すぐ必須ではないが、**仕組み・品質・データ**の改善につながること

## 出力形式

次の **JSON 配列のみ** を出力してください（**必ず3要素**）。

```json
[
  {
    "priority": "high",
    "priorityLabel": "優先度：高",
    "problem": "問題の説明（1〜2文）",
    "action": "• アクション1\n• アクション2\n• アクション3"
  },
  {
    "priority": "medium",
    "priorityLabel": "優先度：中",
    "problem": "…",
    "action": "• …\n• …"
  },
  {
    "priority": "low",
    "priorityLabel": "優先度：低",
    "problem": "…",
    "action": "• …\n• …"
  }
]
```

## 注意

1. **必ず3件**。**JSON 以外の説明文は出力しない**（コードフェンスのみ可）。
2. アクションは「誰が・何を・いつまでに」の粒度で、抽象的な「検討する」だけにしない。
3. 数値（生産額・件数・先週比）を引用する場合は、入力コンテキストの値と矛盾させない。
